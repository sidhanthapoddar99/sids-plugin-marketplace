# Plugin Anatomy

The shape of a plugin on disk and the manifest that describes it. Everything in this chapter is either authored by a plugin maintainer or generated by a scaffolder; nothing here is dynamic at runtime.

The plugin manifest at `.claude-plugin/plugin.json` is the single source of truth for what a plugin is. Every other file is either auto-discovered from a conventional folder or pointed at by a manifest field.

## What's covered

| File | Topic |
|---|---|

| [`02_manifest-fields.md`](./02_manifest-fields.md) | Exhaustive `plugin.json` field reference — identity, component path overrides, modern fields |

| [`04_user-config.md`](./04_user-config.md) | The `userConfig` shape (custom, not JSON Schema), substitution via `${user_config.KEY}`, `CLAUDE_PLUGIN_OPTION_<KEY>` env vars, sensitive-value storage |
